# v2.1.7 - 加载体验优化

**日期**：2026-04-02

## 问题背景

之前的加载流程：
1. 加载基金数据和指数数据
2. 等待建议计算完成
3. 最后一起显示

建议计算需要调用 `pingzhongdata` 接口获取历史净值数据，接口响应较慢，导致用户等待时间长，看不到估算涨跌数据。

## 解决方案

优化后的加载流程：
1. 加载基金数据和指数数据
2. **立即显示估算涨跌**
3. 后台异步计算建议
4. 建议计算完成后更新显示

## 变更内容

### App.vue
- 新增 `adviceLoading` 状态，用于追踪建议加载状态
- `loadData` 函数拆分：基金数据和指数数据优先加载
- 建议数据异步加载，不阻塞主流程

### FundTable.vue
- 新增 `adviceLoading` prop
- 建议和原因列显示"计算中..."加载提示
- PC 端和移动端同步支持

### 代码质量
- 删除未使用的 `useKDJ.js` 组合式函数
- 补充 API 模块详细注释
- 移除 `storage.js` 中未使用的函数
- 移除 `useAdvice.js` 中未使用的参数

## 效果

- 用户可以更快看到基金估值数据
- 建议计算在后台进行，不影响主要信息展示
- 提升了整体用户体验
