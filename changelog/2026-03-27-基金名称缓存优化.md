# 2026-03-27 基金名称缓存优化 + CORS 修复

## 问题描述

1. 管理基金列表弹窗中获取基金名称时，直接调用东财 API 可能因 CORS 或限流导致"网络繁忙"提示。
2. `pingzhongdata` 接口在生产环境存在 CORS 跨域问题，fetch 请求被浏览器阻止。

## 解决方案

### 问题1：基金名称缓存

利用已有的基金数据请求结果缓存基金名称映射 `{ code: name }`，管理基金弹窗直接使用缓存，避免额外的 API 请求。

### 问题2：pingzhongdata CORS 修复

改用 script 标签加载 `pingzhongdata` 接口，该接口返回 JS 文件格式，会设置全局变量 `fS_name` 和 `Data_netWorthTrend`。

## 变更内容

### 1. useFunds.js 添加名称缓存

```javascript
// 基金名称缓存：{ code: name }
const fundNameMap = ref({})

// 在 loadFunds 成功后更新缓存
funds.value.forEach(f => {
  if (f.FCODE && f.SHORTNAME) {
    fundNameMap.value[f.FCODE] = f.SHORTNAME
  }
})

return { funds, fundNameMap, ... }
```

### 2. App.vue 传递缓存

```vue
<FundManageModal
  :fund-name-map="fundNameMap"
  ...
/>
```

### 3. FundManageModal.vue 使用缓存

```vue
<!-- 模板 -->
<span class="fund-name">{{ props.fundNameMap[code] || '--' }}</span>

<!-- 移除 -->
- loadFundNames() 函数
- loadSingleFundName() 函数
- fundNames ref
```

### 4. funds.js pingzhongdata 改用 script 标签

```javascript
export async function fetchPingzhongdata(code) {
  return new Promise((resolve, reject) => {
    const script = document.createElement('script')
    const isDev = typeof import.meta !== 'undefined' && import.meta.env?.DEV
    const baseUrl = isDev ? '/api/eastmoney-fund' : 'https://fund.eastmoney.com'
    script.src = `${baseUrl}/pingzhongdata/${code}.js?v=${Date.now()}`

    script.onload = () => {
      const name = window.fS_name || '--'
      const trend = window.Data_netWorthTrend || []
      // 清理全局变量
      resolve({ name, trend })
    }
    document.head.appendChild(script)
  })
}
```
