# 2026-03-27 基金名称缓存优化

## 问题描述

管理基金列表弹窗中获取基金名称时，直接调用东财 API `fundmobapi.eastmoney.com`，在生产环境可能因 CORS 或限流导致"网络繁忙"提示。

## 解决方案

利用已有的基金数据请求结果缓存基金名称映射 `{ code: name }`，管理基金弹窗直接使用缓存，避免额外的 API 请求。

## 变更内容

### 1. useFunds.js 添加名称缓存

```javascript
// 基金名称缓存：{ code: name }
const fundNameMap = ref({})

// 在 loadFunds 成功后更新缓存
funds.value.forEach(f => {
  if (f.FCODE && f.SHORTNAME) {
    fundNameMap.value[f.FCODE] = f.SHORTNAME
  }
})

return { funds, fundNameMap, ... }
```

### 2. App.vue 传递缓存

```vue
<FundManageModal
  :fund-name-map="fundNameMap"
  ...
/>
```

### 3. FundManageModal.vue 使用缓存

```vue
<!-- 模板 -->
<span class="fund-name">{{ props.fundNameMap[code] || '--' }}</span>

<!-- 移除 -->
- loadFundNames() 函数
- loadSingleFundName() 函数
- fundNames ref
```
